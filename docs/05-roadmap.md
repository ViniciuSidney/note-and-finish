# Roadmap

## Versão atual

### v0.2 - Experiência de uso e fluxo rápido

Status: **Fechada**

Objetivo:
Tornar o uso diário da aplicação mais agradável, rápido e orientado, sem inflar a interface com funcionalidades grandes demais.

Ideia central:
A v0.2 reduziu atrito, melhorou feedbacks visuais e ajudou o usuário a entender rapidamente o que precisa fazer primeiro.

Funcionalidades entregues:
- Painel **Foco de hoje** com atividades atrasadas, para hoje e de alta prioridade.
- Painel **Indicadores** com visão rápida da agenda.
- Painéis do topo recolhíveis.
- Redesign da tela inicial com foco na listagem.
- Botão **Nova atividade** reposicionado no painel de atividades.
- Blocos de prazo refinados visualmente.
- Sistema de toasts.
- Toasts visíveis acima de modais.
- Ações rápidas de prazo com `+1 Dia` e `+1 Semana`.
- Botões de adiamento exibindo a nova data dinamicamente.
- Rolagem automática até o card após adiamento.
- Atalhos de data no modal de criação/edição.
- Adição de etapa ao checklist em tarefa existente.
- Exclusão de etapa do checklist.
- Prévia inteligente do checklist, priorizando etapas pendentes.
- Link para detalhes quando houver etapas ocultas na prévia.
- Progresso compacto do checklist com `✅`.
- Empty states contextuais.
- Modal de detalhes mais horizontal em telas largas.
- Modal de criação reorganizado em layout 30/70 no desktop.
- Ajustes de responsividade do modal de criação.
- Campo visual **Matéria** renomeado para **Categoria**.
- Favicon da aplicação.
- Documentação atualizada para fechamento da versão.

Critérios de fechamento:
- Painel Foco de hoje funcionando com dados reais.
- Ações de adiamento funcionando corretamente.
- Etapas podendo ser adicionadas e removidas em tarefas já criadas.
- Toasts funcionando sem atrapalhar o fluxo.
- Empty states claros e úteis.
- Modal de detalhes mais legível.
- Modal de criação responsivo.
- Testes manuais principais concluídos.

Fora do escopo da v0.2:
- Login.
- Sincronização em nuvem.
- Calendário completo.
- Notificações reais do navegador.
- Sistema complexo de recorrência.
- Relatórios avançados.
- Histórico completo.
- Data de conclusão de tarefa/etapas.
- Customização dos tipos.
- Modo Foco dedicado.

## Próximas versões

### v0.3 - Histórico e acompanhamento simples

Status: Planejada

Objetivo:
Aumentar a utilidade da aplicação para acompanhar mudanças importantes nas atividades ao longo do tempo.

Funcionalidades planejadas:
- Histórico simples de alterações por atividade.
- Registro de conclusão/reabertura.
- Registro de alterações importantes, como prazo, status e prioridade.
- Registro de adição, remoção e conclusão de etapas do checklist.
- Data de conclusão geral da tarefa.
- Data de conclusão de cada etapa.
- Melhor visualização de atividades recentemente modificadas.
- Possível seção de últimas alterações.

Observação:
Essa versão deve continuar simples. O histórico não precisa ser um sistema complexo de auditoria, apenas um apoio para o usuário entender o que mudou.

---

### v0.4 - Dashboard e análise simples

Status: Planejada

Objetivo:
Melhorar a visualização geral do desempenho e da carga de tarefas.

Funcionalidades planejadas:
- Dashboard com distribuição por status.
- Quantidade de atividades por prioridade.
- Quantidade de atividades por tipo.
- Atividades atrasadas em destaque.
- Indicadores simples de progresso.
- Resumo semanal simples.
- Quantidade de atividades concluídas no período.

Observação:
O dashboard deve ser útil para leitura rápida, sem virar uma tela pesada ou cheia de gráficos desnecessários.

---

### v0.5 - Organização avançada

Status: Planejada

Objetivo:
Aprimorar a forma como o usuário organiza atividades por contexto.

Funcionalidades planejadas:
- Melhorias no sistema de etiquetas.
- Possível filtro por categoria com maior destaque.
- Arquivamento de atividades antigas.
- Visualização de atividades por categoria.
- Melhorias no gerenciamento de concluídas.
- Customização das opções do campo Tipo.
- Possível duplicação de tarefa.
- Possíveis templates simples de tarefa.

---

### v0.6 - Experiência mobile e PWA

Status: Planejada

Objetivo:
Melhorar o uso em dispositivos móveis e avaliar instalação como aplicativo.

Funcionalidades planejadas:
- Refinamentos de responsividade.
- Melhorias de toque em botões e menus.
- Ajustes de modais em telas pequenas.
- Manifest PWA.
- Ícones da aplicação.
- Possível funcionamento offline estruturado.

---

### v0.7 - Modo Foco

Status: Ideia planejada

Objetivo:
Criar uma experiência dedicada para priorizar a visualização e execução das atividades, escondendo informações secundárias.

Funcionalidades possíveis:
- Modo Foco ativável pela interface.
- Ocultar painéis de indicadores do topo.
- Destacar a listagem de atividades.
- Possível modal ou tela própria para execução de tarefas prioritárias.

---

### v1.0 - Primeira versão estável

Status: Futura

Objetivo:
Considerar o projeto minimamente completo para uso contínuo pessoal.

Critérios para fechar a versão:
- Funcionalidades principais estáveis.
- Interface consistente em desktop e mobile.
- Backup confiável.
- Código modular e documentado.
- Testes manuais documentados.
- Roadmap das próximas melhorias definido.
- Fluxo principal agradável para uso real.

---
