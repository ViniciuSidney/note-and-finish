Fluxo de uso

Como transformar um template em tela real

O template não é a página final. Ele é a estrutura inicial para copiar, adaptar e conectar com os dados e comportamentos específicos da aplicação.

Escolha o objetivo

Defina se a tela precisa listar, cadastrar, buscar, configurar, focar ou mostrar dados.

Selecione o layout

Use a biblioteca para escolher o molde mais próximo da necessidade real.

Copie o HTML

Pegue o template em `src/templates/layouts` e cole na tela real da aplicação.

Adapte conteúdo e IDs

Troque textos genéricos, remova blocos desnecessários e adicione IDs para o JavaScript.

Conecte a feature

Ligue a tela aos arquivos `controller`, `model`, `service` e `ui` da feature correspondente.

Boas práticas

Regras para manter o modelo limpo

Classes para visual

Use classes para estrutura e aparência. Exemplo: `layout-gallery`, `card`, `btn`, `badge`.

IDs para comportamento

Use IDs somente quando o JavaScript precisar localizar um elemento específico.

Templates não são páginas finais

Eles são moldes copiáveis. A página real deve adaptar conteúdo, dados e comportamento.
